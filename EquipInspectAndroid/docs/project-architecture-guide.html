1. 專案架構總覽（MVVM 分層）

這個 App 是設備巡檢工具：離線登入 → 下載任務 → 掃描巡檢 → 上傳結果。採用 Google 推薦的 MVVM，但 UI 仍用傳統 XML（不是 Jetpack Compose）。

Activity / Fragment + XML Layout + ViewBinding
LoginActivity · MainActivity · 各功能 Fragment
collect StateFlow · 按鈕呼叫 ViewModel 方法

LoginViewModel · DownloadViewModel · InspectViewModel …
管理 UiState、呼叫 Repository、用 Coroutine 做非同步
suspend fun · Result<T>
Repository 層
LocalRepository（Room 本地 DB）
ApiRepository（OkHttp 網路請求）
Entity / DTO / Mapper
資料層
Room SQLite · DataStore 偏好設定 · OkHttp POST JSON
AES 加密 Token · kotlinx.serialization 解析

// sealed class = 有限狀態的 union type（比 enum 更強）
sealed class UiState {
    object Idle : UiState()
    object Loading : UiState()
    data class Error(val message: String) : UiState()
    data class Success(val username: String) : UiState()
}

// when 取代 switch，且會檢查是否 exhaustive
when (st) {
    is UiState.Loading -> setLoading(true)
    is UiState.Error -> toast(st.message)
    is UiState.Success -> navigate()
    else -> { }
}
@HiltViewModel
class LoginViewModel @Inject constructor(
    private val repo: LocalRepository
) : ViewModel() {

    private val _state = MutableStateFlow<UiState>(UiState.Idle)
    val state: StateFlow<UiState> = _state   // 對外唯讀

    fun login(username: String, password: String) {
        viewModelScope.launch {              // 自動隨 ViewModel 生命週期取消
            val user = repo.validateUser(username, password)
            _state.value = if (user != null) UiState.Success(...)
                           else UiState.Error("帳號或密碼錯誤")
        }
    }
}
// Application
@HiltAndroidApp
class MyApp : Application()

// Activity 需要注入時
@AndroidEntryPoint
class MainActivity : AppCompatActivity() {
    private val vm: NavViewModel by viewModels()  // Hilt 自動提供
}

// Module 提供 Singleton
@Module @InstallIn(SingletonComponent::class)
object DatabaseModule {
    @Provides @Singleton
    fun provideLocalRepository(db: AppDatabase) = LocalRepository(db)
}

5. 後端 / 網路語法

API 走 REST POST JSON，驗證靠 Header X-Device-Token（AES 加密的 AndroidId + UserNo）。沒用 Retrofit，直接用 OkHttp。

ViewModel 呼叫 → ApiRepository.upload() → OkHttp POST → ApiParser 解析 Envelope → LocalRepository 更新 Room

API 端點（ApiConfig.kt）

POST /api/app/ping連線測試
POST /api/app/download下載使用者 + 設備 + 異常選項
POST /api/app/upload上傳已巡檢設備

回應格式（統一 Envelope）

{
  "Success": true,
  "Message": "OK",
  "Data": { ... }   // 或數字、陣列，依 API 而定
}

DTO 定義（kotlinx.serialization）

@Serializable
data class ApiEnvelope<T>(
    @SerialName("Data") val data: T? = null,
    @SerialName("Success") val success: Boolean = false,
    @SerialName("Message") val message: String? = null
)

@Serializable
data class EquipDto(
    @SerialName("GUID") val guid: String?,
    @SerialName("Barcode") val barcode: String?,
    ...
)

發送請求（OkHttp 範例）

suspend fun upload(context: Context, repo: LocalRepository): Result<Unit> =
    withContext(Dispatchers.IO) {
        val jsonString = ApiParser.json.encodeToString(UploadBody.serializer(), payload)
        val req = Request.Builder()
            .url(ApiConfig.join(ApiConfig.baseUrl(context), ApiConfig.UPLOAD_PATH))
            .post(jsonString.toRequestBody(ApiConfig.jsonMediaType))
            .addHeader("X-Device-Token", buildDeviceToken(context))
            .build()
        val (code, body) = ApiConfig.client.newCall(req).execute().use { ... }
        ApiParser.parseHttpOrThrow<Int>(code, body)  // 解析 Data 欄位
    }

6. 完整資料流：登入 → 下載 → 巡檢 → 上傳

  1. 登入：LoginViewModel → LocalRepository.validateUser() → Room UserDao（密碼 AES 比對）→ DataStore 存 username
  2. 下載：DownloadViewModel → ApiRepository.download() → 解析 DownloadDto → replaceAllFromDownload() 寫入 Room
  3. 巡檢：InspectViewModel 掃描條碼 → 查 EquipEntity → 更新 abnormal 欄位
  4. 上傳：UploadViewModel → getEquipInspected() → toUploadDto() → POST upload → 成功則 deleteEquipByGuids()
